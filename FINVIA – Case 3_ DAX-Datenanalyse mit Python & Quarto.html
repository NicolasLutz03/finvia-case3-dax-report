FINVIA – Case 3: DAX-Datenanalyse mit Python & Quarto

Author

Nicolas Lutz

1 Zielsetzung

Ziel dieses Use Cases ist die eigenständige Einrichtung eines Python-basierten Analyse-Workflows unter Nutzung von KI-Unterstützung sowie die reproduzierbare Visualisierung historischer Marktdaten.

Untersucht wird der historische Kursverlauf des DAX (Close-Preise) seit 2010.

2 Technische Umsetzung

2.1 Verwendeter Tech-Stack

  • VS Code
  • Python (virtuelle Umgebung via venv)
  • Pakete: yfinance, matplotlib
  • Quarto für reproduzierbare Reports

2.2 Datenbezug & Visualisierung

import yfinance as yf
import matplotlib.pyplot as plt

dax = yf.download("^GDAXI", start="2010-01-01")

plt.figure(figsize=(10, 6))
plt.plot(dax["Close"])
plt.title("DAX – Historischer Kursverlauf (Close)")
plt.xlabel("Datum")
plt.ylabel("Indexstand")
plt.grid(True)
plt.tight_layout()
plt.savefig("dax_close.png", dpi=200)
plt.show()

2.3 3 Beobachtungen

  • Langfristiger Aufwärtstrend des DAX seit 2010
  • Deutliche Volatilität in Krisenphasen (z. B. 2020)
  • Schnelle Erholungsphasen nach Markteinbrüchen

2.4 4 Reflexion: Einsatz von KI

2.4.1 4.1 Vorgehen

  1. Installation von Python
  2. Einrichtung einer virtuellen Umgebung (venv)
  3. Installation der benötigten Pakete (yfinance, matplotlib)
  4. Erstellung eines Analyse-Skripts zur Datenbeschaffung und Visualisierung
  5. Integration des Codes in einen reproduzierbaren Quarto-Report

Der Fokus lag darauf, einen sauberen, nachvollziehbaren Workflow von der Umgebung bis zur finalen HTML-Ausgabe aufzusetzen.

2.4.2 4.2 Rolle der KI

Für dieses Projekt wurde ChatGPT (Version 5.2 Pro) unterstützend eingesetzt.

Die KI half insbesondere bei:

  • Strukturierung eines klaren Minimalbeispiels
  • Unterstützung beim Setup der Python-Umgebung
  • Erklärung der Interpreter-Auswahl in VS Code
  • Fehlerdiagnose (z. B. Unterschied zwischen globalem Python und venv)
  • Anleitung zur Integration in Quarto

Die KI diente primär als technischer Sparringspartner und zur Beschleunigung der Einrichtung.

2.4.3 4.3 Eigene Nachsteuerung

Trotz KI-Unterstützung waren mehrere manuelle Schritte erforderlich:

  • Anpassung der PowerShell Execution Policy
  • Korrekte Aktivierung der virtuellen Umgebung
  • Sicherstellung, dass Pakete im richtigen Interpreter installiert wurden
  • Strukturierung des Reports (YAML-Header, Code-Chunks)
  • Validierung der reproduzierbaren Ausführung

Insbesondere das Zusammenspiel von VS Code, venv und Quarto erforderte eigenständige Fehleranalyse.

2.4.4 4.4 Einordnung des Tech-Stacks

  • Python: vertraut
  • Grundlegendes Coding: vertraut
  • Virtuelle Umgebungen (venv): teilweise vertraut
  • Quarto: neu
  • Reproduzierbare Reports mit Code-Ausführung: neu

Der größte Lernfortschritt lag in der sauberen Integration von Code und Dokumentation in einem reproduzierbaren Workflow.

2.4.5 4.5 Persönliche Bewertung

Der Einsatz von KI hat die technische Einstiegshürde deutlich reduziert und die Problemlösung beschleunigt.
Gleichzeitig war ein grundlegendes Verständnis der Umgebung notwendig, um Fehler richtig einordnen und korrigieren zu können.

Die Kombination aus eigenständigem Setup und KI-Unterstützung erwies sich als effizienter und strukturierter Arbeitsansatz.
